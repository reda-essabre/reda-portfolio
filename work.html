CASE-001 / SQL · Reporting · Data Engineering Closed ✓
Funding Category Extraction Pipeline
Problem Student funding categories stored as nested JSON fields inside a multi-table schema. Analysts extracted data manually each week — 3 hours of work prone to human error and version inconsistency.
Approach Designed a SQL extraction query using JSON path functions, multi-table joins, and conditional aggregation. Built a parameterised report connected directly to the dashboard output layer.
Outcome Zero manual extraction. Report runs on demand or on schedule. Analysts redirected from data wrangling to interpretation. Live dashboard always reflects current state.

CASE-002 / Rule Engine · Automation · Workflows Closed ✓
Automated Late Fee & Account Hold System
Problem Account holds and late fee processing applied manually by staff. Inconsistent timing led to disputes, missed fees, and compliance risk. High volume created daily operational burden.
Approach Mapped all business rules into a structured rule-set design. Configured event-triggered automation: funding change → trigger evaluation → apply action → log. Full audit trail maintained.
Outcome Zero manual intervention for standard cases. Consistent, time-accurate application of fees and holds. Staff time freed from routine enforcement tasks.

CASE-003 / SQL · Performance · Dashboards Closed ✓
Query Optimisation — Overdue Reporting Dashboard
Problem Overdue report used by operations team was timing out under load. Complex multi-join query with no indexing strategy — load time 45+ seconds.
Approach Profiled query execution plan. Identified cross-product join and redundant subquery. Restructured joins, added targeted indexes, introduced CTEs to break down logic.
Outcome Load time: 45s → under 3s. Dashboard refreshes in real-time. Report adopted by additional teams due to reliability improvement.

CASE-004 / n8n · API · Integration Closed ✓
End-to-End Client Onboarding Automation
Problem New client onboarding required manual data entry across 3 systems, email drafts, and calendar scheduling. Average completion time: 2–3 days.
Approach Built n8n workflow triggered by form submission. Data routed to CRM, project tool, and email system via API. Conditional logic for client tier, automated welcome sequence.
Outcome Onboarding time: 2–3 days → under 10 minutes. Zero manual data entry. Consistent communication regardless of staff availability.

CASE-005 / FastAPI · PostgreSQL · AI · Event Systems IN PROGRESS ◆
AI Data-Ops Platform — Personal Engineering Project
Scope 9-week senior-grade engineering project. Full system design from event ingestion to LLM integration, with production-grade reliability patterns throughout.
Architecture FastAPI backend → PostgreSQL → event queue with retry/idempotency → validation layer → RAG-enabled LLM module → observability dashboard.
Key Patterns Idempotent event processing. Retry with exponential backoff. Audit tables. Prompt templates + cost tracking. Structured metrics and alerting.
